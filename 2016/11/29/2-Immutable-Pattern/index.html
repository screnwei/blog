2.Immutable Pattern

别名

  • 想破坏它也没办法

背景

  • 多个线程共享一个实例，实例的状态不会改变。

问题

  • 使用Single Threaded Execution Pattern，会降低 throughpot。

解决方式

  • 当实例建立后状态就不会再变化时，就要停止使用 Single Threaded Execution了。为了避免失误造成更改了实例的状态，故将类写成无法由线程更改。另外，删除实例里所有用来更新状态的方法(setter)。引用实例状态用的方法(getter)就无妨。记得在帮助文档中注明这是一个Immutable的类。


实现

  • 在Java中使用private来隐藏字段，此外由于无法确保不可更改，因此还要使用final。

    /**
 * Created by renwei on 2016/11/18.
 */
public final class Person {
    private final String name;
    private final String address;

    public Person(String name, String address) {
        this.name = name;
        this.address = address;
    }

    public String getName() {
        return name;
    }

    public String getAddress() {
        return address;
    }

    @Override public String toString() {
        return "[ Person: name =" + name + ", address =" + address + "]";
    }
}

相关

  • 对多个线程进行共享互斥，可使用Single Threaded Execution Pattern。当修改用的线程数量比用来读取的线程数量多时，可以考虑使用Read-Write Lock Pattern。
